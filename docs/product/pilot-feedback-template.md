# Pilot Feedback Template

**Status:** In progress — being trialled with real paddlers; feedback received and mostly incorporated.
**Last updated:** 2026-07-03

Use this to run and capture a feedback session with a real paddler on a river
they know. Note: RiverLaunch does **not** seed sections — the app shows the
canonical river, live levels, POIs (access/hazards/features) and amenities, and
**sections are community-created**. So a session validates the river-level view
and the contribution flow, and may include the paddler *creating* a section.

## Demo Script

1. Ask the participant to choose a river/stretch they know.
2. Ask what they would check before paddling it.
3. Ask them to inspect the map, live level/gauge context, hazards, access
   points, features, and amenities.
4. Ask them to add one real piece of local knowledge using the contribution
   flow (hazard / access / feature / report / photo).
5. Ask them to add the stretch they described as a **section** (suggest-a-section),
   and whether that felt natural.
6. Ask what would make them trust or distrust the information.
